## Corrections made when the new benches met a real simulator
- tb_game_doom3d expected the post-win position to be the exit cell's
  entry edge (posx 0xD00). In hardware the win latch gates movement
  from the *following* frame, so the walk carries one 0x40 step into
  the cell before freezing (posx 0xD40, then bit-identical across all
  forced post-win frames -- the freeze itself is exact). The bench's
  two expected constants were corrected to the RTL's behaviour; no DUT
  logic was changed.
- run_all.sh judged a bench by its literal last output line; current
  vvp appends a `$finish called at ...` diagnostic after the bench's
  verdict, so the runner now keys on the last `PASS:`/`FAIL:` line
  instead. No bench logic was changed.
